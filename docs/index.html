Privacy by Design

Your data never leaves your machine.

  • Runs locally only — no cloud uploads or telemetry
  • Makes no external HTTP calls with your member data
  • Binds to localhost only
  • Data stays in memory while loaded

The backup file contains personal data (names, addresses, phone numbers, emails) and must be protected accordingly.

Which one should I download?
  • Windows PC? Pick Windows.
  • Mac, not sure which kind? Click the Apple menu (🍎, top-left of your screen) → About This Mac. If the "Chip" line says Apple M1 / M2 / M3 / M4, you have an Apple Silicon Mac. If it says Intel, you have an Intel Mac. Most Macs sold from late 2020 onwards are Apple Silicon.
  • Mac — Installer vs Portable? Almost everyone wants the Installer (.dmg) — it's the normal "drag to Applications" experience. The Portable (-mac.zip) is mainly used by auto-update and advanced users.
  • What's a .blockmap? It isn't shown here because you don't need it — it's small auto-update metadata used by the app itself.

Getting Started

  1. Download the installer for your operating system from above
  2. Run the installer and follow the prompts
  3. Open your Beacon backup file (`.xlsx`) with the app
  4. The app extracts the backup date from the filename
  5. Choose an analysis to explore your data

Windows users: Because the installer is not yet code-signed, Windows SmartScreen and third-party antivirus tools (Avast, AVG, Norton, McAfee, etc.) may warn that the file is "suspicious" or block the download outright. This is a generic warning given to any unsigned application — it does not mean the file is unsafe.

  • If your browser blocks the download, choose "Keep" / "Keep anyway".
  • If SmartScreen warns on first run, click "More info" → "Run anyway".
  • If your antivirus blocks the installer, choose "Open anyway" / "Allow" — or temporarily allow the file in your AV's quarantine.
  • You can verify the installer by comparing its SHA-256 hash with the one published on the GitHub Releases page, and inspect the source code.

macOS users: the app is not yet code-signed for macOS either, so after dragging it to Applications you may see "Analyse u3a is damaged and can't be opened" on first launch. The file is not actually damaged — macOS is refusing to run an unsigned app downloaded from the internet. To fix it:

  • Open the Terminal app (Applications → Utilities → Terminal).
  • Paste this single line and press Return: 
    xattr -cr "/Applications/Analyse u3a.app"
  • Now double-click "Analyse u3a" in Applications — it will open normally.
  • Make sure you downloaded the right build: Apple Silicon for M1/M2/M3/M4 Macs, Intel for older Macs. See the full macOS install guide if you're unsure.

A code-signing certificate is planned for a future release, which will eliminate most of these warnings on both Windows and macOS.
